# Page near-match label guidance v0

## Scope

This guidance is frozen from the first `30` expert-reviewed, high-overlap,
cross-organization body-page candidates. It governs the next calibration pass
and prevents label drift.

The label describes the best explanation for the observed similarity. It does
not describe legal ownership and does not by itself prove copying.

## Decision Order

1. Identify whether the page is generated from a known external form or
   software product.
2. Identify whether the shared material is an equipment/product technical
   artifact used by both projects.
3. Only after excluding those explanations, consider direct non-normative reuse.
4. Use `uncertain` rather than forcing a conclusion.

High similarity, rare shared text, identical table layout or identical images
are evidence to inspect, not sufficient conditions for `borrowing_candidate`.

## Labels

### `normative_form`

Use when both pages are instances of the same manufacturer, regulatory or
industry form, while project-specific fields or selected equipment differ.

First-batch examples:

- CNP pump-order/product forms with the same layout and manufacturer content;
- heat-exchanger manufacturer forms with different customer/equipment values.

Manufacturer logos, watermarks and repeated product descriptions belong to the
form unless there is independent evidence that one project copied the other's
filled instance.

### `estimate_boilerplate`

Use for pages generated by estimate software or recurring estimate templates.

First-batch examples:

- first pages of local estimate calculations produced by GRAND-Smeta;
- recurring estimate tables with the same columns and frames but different
  project values.

This label is more specific than `normative_form` because estimate-system
output is a distinct and highly recurrent document source.

### `shared_technical_content`

Use when the projects contain the same type of technical artifact, but the
content varies with equipment or engineering parameters.

First-batch examples:

- pump performance diagrams in a common manufacturer frame;
- curves differing because equipment models and operating parameters differ.

This is not the same as `normative_form`: the technical graph or product data is
the meaningful shared content, not merely the surrounding form.

### `borrowing_candidate`

Use only when the pages share substantial non-normative, project-specific
content or composition that is not adequately explained by:

- manufacturer forms;
- regulations or industry standards;
- estimate/software output;
- independently used product documentation;
- the same technical task.

This is a research routing label for deeper investigation, not a final claim.

### `false_positive`

Use when the structural shortlist looks similar numerically but expert review
finds no materially meaningful common form or content.

### `uncertain`

Use when evidence is mixed, page rendering is broken, provenance cannot be
determined or the distinction between labels cannot be made confidently.

## Confidence

- `high`: the source/type of similarity is directly identifiable;
- `medium`: the explanation is most likely but provenance is not explicit;
- `low`: provisional judgment; should be reviewed again.

## First-Batch Result

```text
normative_form:            13
estimate_boilerplate:      13
shared_technical_content:   4
borrowing_candidate:        0
```

The first batch is therefore a negative-control calibration set for borrowing:
even extremely strong cross-organization near-matches can be fully explained
by shared external forms, software templates and common technical material.
