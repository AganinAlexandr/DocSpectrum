# Валидационный набор: УУиР, Подольск, капитальный ремонт

## 1. Назначение набора

Этот набор выбран как первый практический корпус для проверки моделей `section_passport` и `comparison_result` в проекте `DocSpectrum`.

Набор полезен тем, что содержит близкие между собой объекты капитального ремонта, выполненные одной проектной организацией, но с различиями разного масштаба:

- разные жилые дома;
- два близких подраздела документации: `ОВ` и `ГВС`;
- повторяющиеся архитектурные типы домов;
- ожидаемые различия в тексте и графике при большой общей части проектного решения;
- будущий третий слой: версии ПД после экспертизы с корректировками.

## 2. Источник

Исходный файл со списком объектов:

- `E:/commons/DocSpectrum/Капремонт_дома_УУиР_Подольск.xlsx`

Каталог с проектной документацией до экспертизы:

- `E:/MSE_арх`

Количество объектов:

- всего: `35`
- `ОВ`: `20`
- `ГВС`: `15`

Все 35 объектов из Excel найдены в `E:/MSE_арх` по архивному ключу вида `1719_25`.

## 3. Предметная область

Объекты относятся к проектированию `УУиР` - узлов управления и регулирования в жилых домах.

В рабочем понимании:

- `ОВ` относится к системе отопления;
- `ГВС` относится к системе горячего водоснабжения;
- значительная часть оборудования и проектных решений близка;
- документация разделена на два подраздела, что даёт естественную пару близких, но не идентичных разделов.

Для `DocSpectrum` это особенно ценно:

- можно сравнивать один дом в разрезе `ОВ` и `ГВС`;
- можно сравнивать разные дома внутри одного подраздела;
- можно сравнивать архитектурно близкие дома;
- можно позднее добавить слой "до экспертизы / после экспертизы".

## 4. Наблюдение по структуре папок

В папках объектов обнаружены:

- архив `.7z` с проектной документацией;
- XML пояснительной записки.

Пример пары одного адреса:

- `1725_25 Подольск, мкр. Климовск, ул. Симферопольская, 11 (3.10)` - `ОВ`
- `1724_25 Подольск, мкр. Климовск, ул. Симферопольская, 11 (3.11)` - `ГВС`

В XML пояснительной записки для этой пары указана одна проектная организация:

- `Общество с ограниченной ответственностью «Региональная Строительная Производственная Компания»`

## 8. Следующий практический шаг

Следующий шаг - выбрать 1-2 пары `ОВ/ГВС` одного адреса и проверить, какие данные можно извлечь без раскрытия полного архива, а какие потребуют обработки `.7z`.

Практическая рекомендация:

- для первого прохода предварительно распаковать только выбранный малый ряд в `E:/output/DocSpectrum`;
- исходные архивы в `E:/MSE_арх` не изменять;
- в паспортах и результатах сравнения сохранять ссылки на исходный архив и на рабочую распакованную копию;
- после стабилизации pipeline решить, нужна ли автоматическая распаковка архивов на лету.

Для первой пары одного дома использовать:

- в `1724_25` смотреть подпапку `ПД Симф 11 ГВС`;
- в `1725_25` смотреть подпапку `ПД Симф 11 ОТ`.

После этого можно определить первый минимальный pipeline:

- чтение метаданных из XML;
- поиск и извлечение разделов из архива;
- построение минимального `section_passport`;
- построение первого `comparison_result` для пары одного дома.

## 9. Дополнительный слой сравнения: спектр по разделам

Помимо сравнения отдельных подразделов `ОВ` и `ГВС`, для объекта полезно анализировать набор разделов проектной документации целиком.

Минимально интересные разделы:

- `АР`
- `КР`
- `ПОС`
- `ЭС`
- `ОВ`
- `СС`

Такой уровень нужен для оценки относительной доли изменений по разделам.

Например, для двух близких объектов можно увидеть:

- какие разделы почти не меняются;
- какие разделы дают основной вклад в отличие объектов;
- как меняется относительный спектр изменений между архитектурно похожими домами;
- как меняется этот спектр после прохождения экспертизы.
