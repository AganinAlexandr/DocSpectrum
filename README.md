# DocSpectrum

DocSpectrum - проект спектрального и текстового сравнения проектных разделов и документов на базе библиотеки ранее выполненных материалов.

## Суть проекта

DocSpectrum предназначен для анализа проектной документации с двух основных точек зрения:

- спектральное сравнение по составу, количеству и структуре элементов;
- текстовое сравнение по формулировкам, смыслу и устойчивым фрагментам.

Проект должен позволять:

- сравнивать новый раздел с библиотекой ранее выполненных разделов;
- находить ближайшие аналоги и вероятные источники заимствований;
- оценивать долю шаблонных и долю самостоятельных частей;
- сравнивать версии одного раздела;
- формировать паспорт раздела как базовую единицу хранения и анализа.

## Текущий статус

Сейчас в репозитории размещены стартовые проектные документы:

- черновик ТЗ;
- словарь терминов;
- начальная структура репозитория для дальнейшего проектирования схем, заметок и примеров данных.

Следующий крупный этап:

- детальное проектирование логической модели `паспорта раздела`;
- детальное проектирование логической модели `результата сравнения`;
- только после этого - переход к формальным JSON-схемам и реализации.

## Структура репозитория

- `docs/` - ТЗ, словари и основные проектные документы
- `schemas/` - проектирование логических моделей и будущих JSON-схем
- `notes/` - рабочие заметки, решения и промежуточные наблюдения
- `samples/` - примеры структур данных, тестовых описаний и эталонных артефактов

## Инфраструктурная схема

Проект организован в общей трёхслойной инфраструктуре:

- `E:/commons` - общие данные, знания и паспорта алгоритмов
- `E:/repos/DocSpectrum` - версионируемый код и документы проекта
- `E:/output/DocSpectrum` - артефакты прогонов, сравнения и аналитические результаты

## Ближайшие шаги

1. Зафиксировать логическую модель `паспорта раздела`.
2. Зафиксировать логическую модель `результата сравнения`.
3. Определить состав первых `элементов` и `типовых групп элементов` для MVP.
4. Подготовить первые примеры структур данных в `samples/`.
5. После согласования моделей - перейти к реализации.
